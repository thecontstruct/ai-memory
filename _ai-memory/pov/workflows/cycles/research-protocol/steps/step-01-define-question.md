# Step 1: Define the Question Precisely

## STEP GOAL
Vague questions produce vague answers. Before any research begins, Parzival must articulate the exact question being asked using the question definition template. Do not begin research until the template is complete.

## MANDATORY EXECUTION RULES
- Read the complete step file before taking any action
- Follow the sequence exactly as written
- Do not skip or reorder steps

## CONTEXT BOUNDARIES
- Available context: The uncertainty that triggered this protocol, the context in which it arose (review, implementation, planning)
- Limits: Do not begin searching for answers at this stage. This step is question formulation only.

## MANDATORY SEQUENCE

### 1. Complete the Question Definition Template

**RESEARCH QUESTION:**
What specifically needs to be known?
[One specific, unambiguous question -- not a topic area]

**WHY IT MATTERS:**
What decision or action depends on this answer?
[The downstream impact of getting this wrong]

**WHAT IS ALREADY KNOWN:**
What project files have been checked?
What do they say (or not say) about this?

**WHAT WOULD RESOLVE IT:**
What would a satisfactory answer look like?
What sources would be trusted for this answer?

**ACCEPTABLE ANSWER FORMS:**
- A project file citation that directly addresses this
- Official documentation for [specific technology]
- A verified best practice with a named, credible source
- A user decision with reasoning documented

### 2. Verify Question Quality
Before proceeding, confirm:
- The question is specific and unambiguous
- The downstream impact is clear
- What has already been checked is documented
- Acceptable answer forms are identified

A poorly defined question wastes research cycles and produces unreliable answers. Refine until the question is precise.

## CRITICAL STEP COMPLETION NOTE
ONLY when the question definition template is complete and the question is specific and unambiguous, load and read fully {nextStepFile}

## SYSTEM SUCCESS/FAILURE METRICS

### SUCCESS:
- Question is specific (one question, not a topic area)
- Downstream impact is clearly articulated
- Prior knowledge and gaps are documented
- Acceptable answer forms are identified

### FAILURE:
- Vague or multi-part question
- Beginning research without completing the template
- Not documenting what has already been checked
- Not identifying what a satisfactory answer looks like
