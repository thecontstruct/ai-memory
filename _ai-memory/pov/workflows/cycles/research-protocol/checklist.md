# Cycles Research Protocol — Validation Checklist

## Pre-Execution Checks

- [ ] Research question is triggered by a genuine knowledge gap (not a shortcut)
- [ ] Layers 1 and 2 have not been skipped

## Step Completion Checks

### Step 1: Define Question (step-01-define-question)
- [ ] Question is specific (one question, not a topic area)
- [ ] Downstream impact is clearly articulated
- [ ] Prior knowledge and gaps are documented
- [ ] Acceptable answer forms are identified

### Step 2: Layer 1 — Project Files (step-02-layer1-project-files)
- [ ] All five file categories searched in order
- [ ] Findings recorded with specific citations
- [ ] Clear determination of FOUND / PARTIAL / NOT FOUND
- [ ] No skipping ahead to external sources

### Step 3: Layer 2 — Documentation (step-03-layer2-documentation)
- [ ] Exact technology version identified before searching
- [ ] Sources consulted in priority order
- [ ] Findings include full citations with source, version, section
- [ ] Answer verified against project-specific context
- [ ] Conflicts documented when found

### Step 4: Layer 3 — Analyst Research (step-04-layer3-analyst-research)
- [ ] Analyst was activated only after Layers 1-2 were exhausted
- [ ] Research instruction included all prior layer findings
- [ ] Analyst output was independently verified before acceptance
- [ ] All findings include specific file paths and line numbers
- [ ] Answer resolves the original research question

### Step 5: Escalate to User (step-05-escalate-to-user)
- [ ] Escalation includes complete research trail from all three layers
- [ ] Options are presented with concrete trade-offs specific to this project
- [ ] Parzival's recommendation is stated (or absence of recommendation is explained)
- [ ] User decision is recorded with reasoning
- [ ] Decision will be documented in project files

### Step 6: Document Answer (step-06-document-answer)
- [ ] Answer is documented in the appropriate project file
- [ ] Documentation includes source, reasoning, and applicability
- [ ] Confidence level is assigned correctly
- [ ] Research log entry is complete
- [ ] User decisions are confirmed as accurately documented

## Workflow-Level Checks

- [ ] Layers executed in order (1 → 2 → 3 → escalation)
- [ ] Answer documented in the appropriate project file
- [ ] Research log entry complete
- [ ] No SYSTEM FAILURE conditions triggered in any step

## Anti-Pattern Checks

- [ ] Did NOT skip Layer 1 and go straight to external docs
- [ ] Did NOT begin research without completing the question template
- [ ] Did NOT skip any of the five file categories in Layer 1
- [ ] Did NOT accept partial answers without evaluating sufficiency
- [ ] Did NOT use sources from a different version of the technology
- [ ] Did NOT accept blog posts or forum answers as authoritative
- [ ] Did NOT stop when an answer "sounds right" instead of when it is verified
- [ ] Did NOT activate Analyst before exhausting Layers 1-2
- [ ] Did NOT accept Analyst output without independent verification
- [ ] Did NOT accept output with unsupported assumptions
- [ ] Did NOT escalate to user before exhausting all three layers
- [ ] Did NOT present vague options without specific trade-offs
- [ ] Did NOT proceed without user decision when escalation was reached
- [ ] Did NOT fail to document the answer after finding it
- [ ] Did NOT document in the wrong project file
- [ ] Did NOT proceed on UNCERTAIN confidence
